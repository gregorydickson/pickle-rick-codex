## Trap Doors

- `detached-launch.js` — INVARIANT: empty fresh launch/cwd locks keep the write grace; non-empty non-PID payloads are corrupt and reclaimable. BREAKS: relaunches either steal live reservations or wedge behind junk lockfiles. ENFORCE: `advanced loop resume refuses to relaunch while a fresh empty tmux launch lock is held`; `pickle-tmux clears a fresh malformed launch lock before relaunching`. PATTERN_SHAPE: `fs.openSync(lockPath,'wx')` EEXIST handlers that apply the empty-file grace to any unreadable payload.
- `git-utils.js` — INVARIANT: detached auto-commits must ignore baseline `??` files and stage only tracked edits plus iteration-created untracked paths. BREAKS: pre-existing scratch files block clean ticket commits or get swept into detached ticket history. ENFORCE: `mux-runner preserves pre-existing untracked files while auto-committing tracked ticket changes`. PATTERN_SHAPE: tmux ticket auto-commit paths branching on whole-tree dirt or calling repo-wide staging without diffing against baseline untracked paths.
- `pipeline-bootstrap.js` — INVARIANT: baseline capture only executes verification for tickets still eligible to run. BREAKS: resume/bootstrap replays side-effectful verification from done, blocked, or skipped tickets. ENFORCE: `ensureBootstrapSessionReady does not rerun verification for terminal tickets when capturing baselines`. PATTERN_SHAPE: baseline capture loops over manifest/summary tickets without filtering terminal statuses.
- `pipeline-state.js` — INVARIANT: when scope-key builders change, stored verification baselines must re-key against session-relative targets or stored scope targets, never ambient `process.cwd()`. BREAKS: resumed sessions lose known unrelated failure baselines and block scoped verification unexpectedly, especially with absolute `--prefix` package tests. ENFORCE: `spawn-morty migrates legacy scoped package-manager baseline keys before subtracting known unrelated failures`; `spawn-morty preserves absolute --prefix scoped baseline keys across session-state normalization`. PATTERN_SHAPE: baseline normalizers rebuilding `scope.key` from command text without the session working dir or persisted targets; INVARIANT: nullable child-process exit codes stay non-zero unless the process returned code `0`. BREAKS: timed-out verification can pass and mark tickets done without verification. ENFORCE: `spawn-morty blocks timed-out verification commands instead of treating signal exits as passing verification`. PATTERN_SHAPE: verification failure builders coercing nullable exit codes through `Number(...)` before the zero-success check; INVARIANT: package-test scopes and parsed node-test failures normalize to the same session-relative path, including `pnpm --filter` package selectors. BREAKS: filtered or prefixed package-manager baselines can stop subtracting unrelated reds or misclassify explicit failing targets. ENFORCE: `spawn-morty still blocks prefixed scoped package-manager test failures when the same identity exists in the baseline`; `spawn-morty preserves pnpm --filter scoped baseline subtraction for package node-test verification`. PATTERN_SHAPE: package-manager scope builders resolving `--prefix` but not `pnpm --filter` against the owning package cwd while failure parsers keep `test at` paths package-relative; INVARIANT: scoped Vitest rewrites must preserve package-test scope and per-file failure identities after `cd <pkg> && <pm> exec vitest run ...`. BREAKS: rewritten scoped verification collapses into generic command failures and stops subtracting known unrelated reds. ENFORCE: `spawn-morty preserves scoped baseline subtraction after rewriting npm --prefix vitest verification`. PATTERN_SHAPE: scope/failure parsers that understand raw package-manager tests but not rewritten `cd ... && ... vitest run` commands; INVARIANT: verification baseline migration and persistence must normalize against `pipeline.working_dir`, not the caller shell cwd. BREAKS: legacy absolute `node --test` baselines are re-keyed outside the session scope and unrelated reds stop subtracting on resume. ENFORCE: `spawn-morty migrates legacy absolute node --test baseline keys against the session working dir`. PATTERN_SHAPE: baseline read/write helpers calling `normalizeVerificationBaselines(...)` without the pipeline contract cwd.
- `progress-snapshot.js` — INVARIANT: progress diffs must count explicit step or ticket movement even when git/artifacts stay unchanged. BREAKS: healthy ticket/phase advancement trips the circuit breaker as stalled work. ENFORCE: `progress snapshot treats phase and ticket advancement as progress even without file churn`; `circuit breaker treats step and ticket advancement as progress`. PATTERN_SHAPE: snapshot comparators capturing `step`/`current_ticket` but diffing only tree hashes or artifact digests.
- `session-map.js` — INVARIANT: fresh empty map locks keep the startup grace, but fresh non-empty malformed payloads, including JSON objects without valid `pid` and `ts`, are junk and must be reclaimed immediately. BREAKS: `updateSessionMap()` blocks until timeout behind corrupt `current_sessions.json.lock` payloads. ENFORCE: `session-map reclaims fresh malformed non-empty locks before timing out`; `session-map reclaims fresh malformed structured locks before timing out`. PATTERN_SHAPE: session-map lock stealers treating any parsed JSON object as valid before confirming integer `pid` plus finite `ts`.
- `state-manager.js` — INVARIANT: fresh empty locks keep the startup grace, but fresh non-empty malformed payloads, including JSON objects without valid `pid` and `ts`, are junk and must be reclaimed immediately. BREAKS: crashed writers leave malformed state locks that block session updates until the stale window exceeds the lock timeout. ENFORCE: `StateManager update reclaims fresh malformed non-empty lock files`; `StateManager update reclaims fresh malformed structured lock files`. PATTERN_SHAPE: lock stealers treating any parsed JSON object as valid before confirming integer `pid` plus finite `ts`.
- `tickets.js` — INVARIANT: ticket files must exactly match the current manifest, including removing obsolete files when the manifest shrinks or empties. BREAKS: stale statuses or deleted tickets stay runnable and detached execution consumes work the manifest no longer declares. ENFORCE: `ensureTicketFilesMaterialized rewrites stale ticket files when manifest status changes without an id change`; `summarizeTickets and ensureTicketFilesMaterialized prune obsolete ticket files when the manifest shrinks or empties`. PATTERN_SHAPE: materialization guards that compare only ticket ids/cardinality or rewrite current tickets without deleting stale `linear_ticket_*.md`.
- `verification-env.js` — INVARIANT: scoped package-manager test rewrites must resolve the owning package before reading `package.json`, including selectors such as `--prefix` and `pnpm --filter`. BREAKS: `npm|pnpm|yarn|bun test -- <file>` falls back to the broad wrapper and runs the wrong suite. ENFORCE: `spawn-morty rewrites npm --prefix scoped vitest verification commands into targeted execution`; `spawn-morty rewrites pnpm --filter scoped vitest verification commands into targeted execution`. PATTERN_SHAPE: `extractPackageDirFromTokens()` parsers that handle `test -- <file>` but omit manager-specific package selectors such as `--prefix` or `--filter`.
