## Trap Doors

- `detached-launch.js` — INVARIANT: empty fresh launch/cwd locks keep the write grace; non-empty non-PID payloads are corrupt and reclaimable. BREAKS: relaunches either steal live reservations or wedge behind junk lockfiles. ENFORCE: `advanced loop resume refuses to relaunch while a fresh empty tmux launch lock is held`; `pickle-tmux clears a fresh malformed launch lock before relaunching`. PATTERN_SHAPE: `fs.openSync(lockPath,'wx')` EEXIST handlers that apply the empty-file grace to any unreadable payload.
- `pipeline-bootstrap.js` — INVARIANT: baseline capture only executes verification for tickets still eligible to run. BREAKS: resume/bootstrap replays side-effectful verification from done, blocked, or skipped tickets. ENFORCE: `ensureBootstrapSessionReady does not rerun verification for terminal tickets when capturing baselines`. PATTERN_SHAPE: baseline capture loops over manifest/summary tickets without filtering terminal statuses.
- `pipeline-state.js` — INVARIANT: when scope-key builders change, stored verification baselines must be re-keyed on read to the current normalized scope. BREAKS: resumed sessions lose known unrelated failure baselines and block scoped verification unexpectedly. ENFORCE: `spawn-morty migrates legacy scoped package-manager baseline keys before subtracting known unrelated failures`. PATTERN_SHAPE: baseline normalizers that update `scope.key` metadata but leave map entries stored under legacy keys; INVARIANT: nullable child-process exit codes stay non-zero unless the process returned code `0`. BREAKS: timed-out verification can pass and mark tickets done without verification. ENFORCE: `spawn-morty blocks timed-out verification commands instead of treating signal exits as passing verification`. PATTERN_SHAPE: verification failure builders coercing nullable exit codes through `Number(...)` before the zero-success check; INVARIANT: package-test scopes and parsed node-test failures normalize to the same session-relative path. BREAKS: `npm --prefix ... test -- <file>` baseline subtraction can pass explicit failing targets. ENFORCE: `spawn-morty still blocks prefixed scoped package-manager test failures when the same identity exists in the baseline`. PATTERN_SHAPE: package-manager scope builders resolving `--prefix` targets against package cwd while failure parsers keep `test at` paths package-relative.
- `progress-snapshot.js` — INVARIANT: progress diffs must count explicit step or ticket movement even when git/artifacts stay unchanged. BREAKS: healthy ticket/phase advancement trips the circuit breaker as stalled work. ENFORCE: `progress snapshot treats phase and ticket advancement as progress even without file churn`; `circuit breaker treats step and ticket advancement as progress`. PATTERN_SHAPE: snapshot comparators capturing `step`/`current_ticket` but diffing only tree hashes or artifact digests.
- `session-map.js` — INVARIANT: fresh empty session-map locks keep the startup grace, but fresh non-empty non-PID locks are junk and must be reclaimed immediately. BREAKS: crashed writers leave malformed session-map locks that block session recovery until the stale window exceeds the lock timeout. ENFORCE: `session-map reclaims fresh malformed non-empty locks before timing out`. PATTERN_SHAPE: lock stealers checking lock age before distinguishing empty startup locks from non-empty malformed payloads.
- `state-manager.js` — INVARIANT: fresh empty locks keep the startup grace, but fresh non-empty non-PID locks are junk and must be reclaimed immediately. BREAKS: crashed writers leave malformed state locks that block session updates until the stale window exceeds the lock timeout. ENFORCE: `StateManager update reclaims fresh malformed non-empty lock files`. PATTERN_SHAPE: lock stealers checking lock age before distinguishing empty startup locks from non-empty malformed payloads.
- `tickets.js` — INVARIANT: ticket files must be rematerialized whenever manifest content changes, not only when ids/count change. BREAKS: stale file statuses or verification fields let summaries and mux-runner execute tickets the manifest already skipped or finished. ENFORCE: `ensureTicketFilesMaterialized rewrites stale ticket files when manifest status changes without an id change`. PATTERN_SHAPE: materialization guards that compare only ticket ids/cardinality before reusing `linear_ticket_*.md`.
- `verification-env.js` — INVARIANT: scoped package-manager test rewrites must honor package cwd flags before reading `package.json`. BREAKS: `npm|pnpm|yarn|bun test -- <file>` falls back to the broad wrapper and runs the wrong suite. ENFORCE: `spawn-morty rewrites npm --prefix scoped vitest verification commands into targeted execution`. PATTERN_SHAPE: `extractPackageDirFromTokens()` parsers that handle `test -- <file>` but omit manager-specific cwd selectors such as `--prefix`.
